# ADR 0002: Development Workflow

## Status

Accepted.

## Decision

cppFan work should be planned before implementation. Each feature should have a clear goal, acceptance criteria, edit scope, tests, and rollback notes.

## Workflow

- Create issue.
- Read project docs.
- Confirm edit scope.
- Make focused changes.
- Run checks.
- Open pull request.

## Benefits

- Focused changes.
- Easier review.
- Safer iteration.
- Better project memory.
